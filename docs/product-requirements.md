# isearch 产品需求文档

## 1. 产品概述

**产品名称**: isearch
**产品类型**: 本地文件搜索工具
**核心定位**: 快速、准确、易用的本地文件系统搜索引擎

isearch 是一款面向桌面用户的本地文件搜索工具，旨在帮助用户在海量文件中快速定位目标文件。与系统自带搜索相比，isearch 提供更智能的模糊匹配能力和更友好的交互体验。

## 2. 用户故事

| 角色 | 场景 | 需求 |
|------|------|------|
| 开发者 | 查找项目中的某个源文件 | 按文件名模糊匹配，快速定位 |
| 设计师 | 找到上周下载的设计素材 | 按文件名和时间范围筛选 |
| 普通用户 | 找到文档但只记得部分名称 | 模糊搜索，支持拼写容错 |
| 运维人员 | 查找配置文件 | 支持按路径过滤 |

## 3. 功能需求

### 3.1 核心功能 (V1.0)

#### F1: 全文索引
- 对指定目录下的所有文件建立索引
- 索引内容包括：文件名、文件路径、文件大小、修改时间
- 支持增量索引：仅索引新增或修改的文件
- 支持后台索引：不影响用户正常使用

#### F2: 文件搜索
- **精确搜索**: 输入完整文件名，精确匹配
- **模糊搜索**: 支持部分匹配、通配符匹配
- **路径搜索**: 支持按路径筛选结果
- 搜索结果按相关度排序

#### F3: 搜索结果显示
- 显示文件名（高亮匹配部分）
- 显示文件完整路径
- 显示文件大小和修改时间
- 支持分页展示（每页 20 条）
- 支持双击打开文件

### 3.2 扩展功能 (未来版本)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 内容搜索 | 支持搜索文件内容 | P2 |
| 过滤器 | 按类型、大小、日期过滤 | P2 |
| 收藏夹 | 收藏常用搜索条件 | P3 |
| 搜索历史 | 记录最近搜索记录 | P3 |
| 命令行工具 | 提供 CLI 接口 | P3 |

## 4. 非功能需求

### 4.1 性能需求
- 搜索响应时间 < 100ms（10000 个文件以内）
- 索引构建速度 > 1000 文件/秒
- 内存占用 < 200MB（100000 个文件）

### 4.2 可用性需求
- 支持 macOS / Linux / Windows
- 提供图形界面（GUI）和命令行界面（CLI）
- 安装包 < 50MB

### 4.3 可维护性需求
- 模块化设计，便于扩展
- 支持插件机制
- 配置文件格式：TOML / JSON

## 5. 术语表

| 术语 | 定义 |
|------|------|
| 索引 | 文件名和路径的预构建数据结构，用于加速搜索 |
| 模糊搜索 | 支持部分字符匹配的搜索方式，允许一定的误差 |
| 增量索引 | 仅对新增或修改文件进行索引，避免全量重建 |
| 相关度排序 | 根据匹配程度对搜索结果进行排序 |

## 6. 成功标准

- [ ] 用户输入关键词后 100ms 内返回搜索结果
- [ ] 模糊搜索能正确匹配用户意图（如 "doc" 能找到 "document.pdf"）
- [ ] 索引构建过程不阻塞搜索操作
- [ ] 跨平台支持 macOS、Linux、Windows
